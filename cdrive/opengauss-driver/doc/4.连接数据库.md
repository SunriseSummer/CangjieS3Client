## 连接数据库

### 1 普通连接方式

在创建数据库连接之后，才能使用它来执行SQL语句操作数据。

#### 1.1 获取数据源

创建数据库连接前，要先通过驱动Driver对象获取数据源，然后才能通过数据源创建数据库连接。

1. Driver接口获取数据源的函数原型

   ```
   func open(connectionString: String, opts: Array<(String, String)>): Datasource
   
   示例：
   let url = "opengauss://username:password@127.0.0.1:5432/database?sslmode=disable"
   let ds = drv.open(url)
   ```

2. url整体格式

   ```
   scheme:[//user:password@host:port,host2:port2]/database?params
   ```

3. 驱动库提供了把连接串配置到配置文件中的工具类

   1. env.props配置文件

      ```
      # 连接串
      connectionString=opengauss://username:password@127.0.0.1:5432/database?sslmode=disable
      ```

      1. env.props中的内容是 key=value 的格式，支持#行注释。
      2. env.props放在运行程序当前目录或父目录下，比如驱动包根目录自带env.props文件，把文件的内容根据需要修改后，运行samples下的示例或cjpm test执行测试用例，都可以使用这个env.props文件中定义的内容。

   2. 加载env.props配置文件的代码

      ```
      // 导入工具包
      import cangjie_tpc::opengauss.utils.*
      
      main(): Int64 {
        // 调用loadDefaultEnv加载env.props，并设置到环境变量中
        loadDefaultEnv()
        // 通过环境变量获取env.props中的连接串
        let url = getEnvString("connectionString")
      
        ...// 其他代码
      
        // 获取数据源
      
      ```

      

#### 1.2 通过数据源建立连接

1. 数据源获取连接的原型如下

   ```
   func connect(): Connection
   ```

2. 完整示例如下

   ```
   package sqltype_example
     
   import std.database.sql.*
   import cangjie_tpc::opengauss.driver.*
   import cangjie_tpc::opengauss.utils.*
   
   main(): Int64 {
     loadDefaultEnv()
     let url = getEnvString("connectionString")
     let drvopt = DriverManager.getDriver("opengauss")
     var drv: Driver
     match (drvopt) {
         case Some(v) => drv = v
         case None =>
             println("cannot get the opengauss driver")
             return 1
     }
     let ds = drv.open(url)
     // 建立连接，try()目的是为了连接使用完后自动关闭
     try (conn = ds.connect()) {
         var sql = "insert into typetest1(char1) values(?)"
         var statement = conn.prepareStatement(sql)
         var char1 = "09:18:23"
         statement.set<String>(1, char1)
         var updateresult = statement.update()
         statement.close()
     } catch (ex: Exception) {
         ex.printStackTrace()
     }
   
     return 0
   }
   ```



### 2 连接数据库（以SSL方式）

​	用户连接openGauss服务器时，可以通过开启SSL加密客户端和服务器之间的通讯，为敏感数据在Internet上的传输提供了一种安全保障手段。本小节主要介绍应用程序通过仓颉语言如何采用SSL的方式连接openGauss。在使用本小节所描述的方法前，默认用户已经获取了服务端和客户端所需要的证书和私钥文件，关于证书等文件的获取请参考Openssl相关文档和命令。

#### 2.1 服务端配置

当开启SSL模式后，必须提供根证书、服务器证书和私钥。

配置步骤（假设用户的证书文件放在数据目录/gaussdb/data/datanode下，且采用默认文件名）：

1. 以操作系统用户omm登录数据库主节点。

2. 生成并配置证书。

   1. 生成SSL证书，具体请参见证书生成。将生成出的文件server.crt、server.key、cacert.pem拷贝到服务端数据目录下。

   2. 使用如下命令可以查询数据库节点的数据目录，instance列为数据目录。

      ```
      gs_om -t status --detail
      ```

   3. 在Unix系统上，server.crt、server.key的权限设置必须禁止任何外部或组的访问，请执行如下命令实现这一点。

      ```
      chmod 0600 server.key
      ```

3. 开启SSL认证模式。

   ```
   gs_guc set -D /gaussdb/data/datanode -c "ssl=on"
   ```

4. （可选）配置客户端接入认证参数，IP为所要连接的主机IP。

   ```
   gs_guc reload -D /gaussdb/data/datanode -h "hostssl all             all             127.0.0.1/32           cert"
   gs_guc reload -D /gaussdb/data/datanode -h "hostssl all             all             IP/32           cert"
   ```

   > [!NOTE]
   >
   > 不修改这个配置，也可以达到加密通信的目的。
   > 如果服务端pg_hba.conf文件中METHOD配置为cert，则只有客户端使用证书（client.crt）中所设置的用户名（common name）才能够成功连接数据库。如果设置为md5、sm3或sha256则对连接数据库的用户没有限制。
   > MD5加密算法安全性低，存在安全风险，建议使用更安全的加密算法。

5. 配置SSL认证相关的数字证书参数。

   1. 各命令后所附为设置成功的回显。

      ```
      gs_guc set -D /gaussdb/data/datanode -c "ssl_cert_file='server.crt'"
      gs_guc set: ssl_cert_file='server.crt'
      ```

      ```
      gs_guc set -D /gaussdb/data/datanode -c "ssl_key_file='server.key'"
      gs_guc set: ssl_key_file='server.key'
      ```

6. 重启数据库。

   ```
   gs_om -t stop && gs_om -t start
   ```

   

#### 2.2 客户端配置

1. 在连接串中配置sslmode=require启用以SSL方式连接数据库

   ```
   opengauss://username:password@127.0.0.1:5432/bes?sslmode=require
   ```

> [!NOTE]
>
> ​	配置sslmode=prefer可以在以SSL方式连接数据库失败时，再通过非SSL方式连接数据库。
