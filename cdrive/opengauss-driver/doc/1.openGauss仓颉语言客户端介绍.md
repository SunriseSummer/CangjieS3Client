## openGauss仓颉语言客户端介绍

### 1.1 介绍

opengauss-driver是纯仓颉语言实现的[openGauss](https://opengauss.org/)和[PostgreSQL](https://www.postgresql.org/)数据库驱动。

### 1.2 特性

1. 通过在连接串中添加逗号隔开的多个数据库实例地址，可以让驱动连接主备集群实例

2. 驱动可以识别主实例，自动把DDL、DML、DCL发给主实例

3. transaction事务会连接主实例来完成

4. 驱动有容错功能，碰到连接不上的实例时，会自动连接下一个正常的实例

5. 支持openGauss和PostgreSQL

   

### 1.3 仓颉基础环境准备

#### 1.3.1 仓颉语言安装

仓颉语言安装包下载地址：https://cangjie-lang.cn/download

##### 1.3.1.1 windows

- 下载windows的压缩包解压后，启动命令行时，先执行解压目录下的envsetup.bat设置环境变量，或者手动设置以下环境变量

  | 变量名       | 操作 | 变量值                                                       |
  | ------------ | ---- | ------------------------------------------------------------ |
  | CANGJIE_HOME | 新建 | 仓颉语言压缩包解压目录的cangjie目录。例如：E:\cangjie        |
  | Path         | 编辑 | 添加仓颉的路径%CANGJIE_HOME%runtime\lib\windows_x86_64_llvm;%CANGJIE_HOME%bin;%CANGJIE_HOME%tools\bin;%CANGJIE_HOME%tools\lib |

##### 1.3.1.2 linux

- 下载linux的压缩包解压后，在解压目录，执行source envsetup.sh设置环境变量，或者手动设置以下环境变量

| 变量名          | 操作                     | 变量值                                                       |
| --------------- | ------------------------ | ------------------------------------------------------------ |
| CANGJIE_HOME    | 新建                     | 仓颉语言压缩包解压目录的cangjie目录。例如：/opt/cangjie      |
| hw_arch         | 新建                     | 根据操作系统配置，比如x86_64，可以参考解压目录的runtime/lib下文件来建立 |
| PATH            | 编辑                     | 添加仓颉的路径\$\{CANGJIE_HOME\}/bin:\${CANGJIE_HOME}/tools/bin |
| LD_LIBRARY_PATH | 不存在则新建，存在则编辑 | 添加仓颉的库路径\${CANGJIE_HOME}/runtime/lib/linux_\${hw_arch}_llvm:\${CANGJIE_HOME}/tools/lib |



#### 1.3.2 仓颉编程语言 stdx 模块安装

仓颉编程语言 stdx 模块下载地址：https://gitcode.com/Cangjie/cangjie-stdx

##### 1.3.2.1 windows

下载windows的压缩包解压后，手动设置以下环境变量

| 变量名       | 操作 | 变量值                                                       |
| ------------ | ---- | ------------------------------------------------------------ |
| CANGJIE_STDX | 新建 | stdx 模块位置。例如：C:\cangjiestdx\cangjie-stdx-windows-x64-1.0.0.1\windows_x86_64_llvm\static\stdx |

##### 1.3.2.2 linux

下载linux的压缩包解压后，手动设置以下环境变量

| 变量名       | 操作 | 变量值                                                       |
| ------------ | ---- | ------------------------------------------------------------ |
| CANGJIE_STDX | 新建 | stdx 模块位置。例如：/opt/install/stdx/linux_x86_64_llvm/static/stdx |



### 1.4 获取opengauss仓颉语言客户端

Opengauss仓颉语言客户端项目托管在gitcode上，项目地址为：
https://gitcode.com/Cangjie-TPC/opengauss-driver 

使用git下载Opengauss仓颉语言客户端：
` git clone https://gitcode.com/Cangjie-TPC/opengauss-driver.git `



### 1.5 通过源码方式引入opengauss客户端依赖

仓颉1.0.0 LTS版本：在项目的cjpm.toml中添加dependencies引入opengauss依赖：

```yaml
[dependencies]
  opengauss = {git = "https://gitcode.com/Cangjie-TPC/opengauss-driver.git", branch = "master"}
```

更新依赖，运行cjpm update会自动下载依赖opengauss项目到~/.cjpm目录下
` cjpm update`



### 1.6 编译Opengauss仓颉语言客户端

清理工程，在工程根目录下运行：
` cjpm clean`

编译工程，在工程根目录下运行：
` cjpm build`



### 1.7  运行源码包中的示例程序

在samples下有示例程序，比如sqltype_example，可以通过如下方式编译运行。

```
###### 运行前需要把驱动包解压根目录的env.props中的connectionString连接串中的数据库地址和用户名密码配置正确。
###### 运行前需要把驱动包解压目录data的simpletest.sql导入到数据库中。
cd sqltype_example
###### 编译
cjpm build
###### 运行
cjpm run
```
