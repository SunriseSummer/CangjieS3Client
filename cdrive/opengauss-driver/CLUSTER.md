# 连接数据库主备集群

通过在连接串中添加逗号隔开的多个数据库实例地址，可以让驱动连接主备集群实例。驱动可以识别主实例，自动把DDL、DML、DCL发给主实例。另外，transaction事务也会连接主实例来完成。另外，驱动有容错功能，碰到连接不上的实例时，会自动连接下一个正常的实例。

例如：数据库主备集群是192.168.0.10:5432（主） 192.168.0.11:5432（备）192.168.0.12:5432（备），那么

连接串可以配置成**opengauss://bes:password@192.168.0.10:5432,192.168.0.11:5432,192.168.0.12:5432/bes?sslmode=disable**

其中主备地址的顺序没有要求，即也可以配置成**opengauss://bes:password@192.168.0.11:5432,192.168.0.10:5432,192.168.0.12:5432/bes?sslmode=disable**

## 集群连接的配置

- balance  均衡策略
  
  - leastconn 最少连接（默认），即会选择当前驱动记录的连接最少的实例地址
  
  - roundrobin 轮询，即轮流选择实例地址

- shuffle_addrs 是否打乱连接地址顺序。
  
  - false 使用连接串中的地址顺序（默认），即遍历实例地址时，按照连接串中的顺序来遍历。
  
  - true 打乱连接串中的地址顺序，即遍历实例地址时，不按照连接串中的顺序来遍历。

- priority_addrs 优先连接的实例地址。格式是逗号隔开的多个服务地址（ip:port,ip:port,...）
  
  假设主备集群是192.168.0.10:5432（主） 192.168.0.11:5432（备）192.168.0.12:5432（备），使用最少连接的均衡策略，打乱连接串地址，优先连接两个备实例，那么连接串的配置是 **opengauss://bes:password@192.168.0.10:5432,192.168.0.11:5432,192.168.0.12:5432/bes?sslmode=disable&shuffle_addrs=false&balance=leastconn&shuffle_addrs=true&priority_addrs=192.168.0.11:5432,192.168.0.12:5432**
